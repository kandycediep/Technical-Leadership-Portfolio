# M1: The Unasked Question

*Creating a resource that solves an existing problem.*

**Target Skills: Resource Creation, Stakeholder Identification, Problem Solving**

## Task

One technical community that I’m a part of is a part-time internship I completed during the school year with a tech startup. For this project, I created a step-by-step tutorial on how to create Splunk dashboards in an intentional and effective way for various audiences.

## Process

This presentation is designed for individuals who are just beginning to learn how to navigate Splunk and create dashboards that communicate complex, real-time data in visual ways for non-technical audiences. When I was first learning Splunk, I remember feeling extremely overwhelmed by the platform’s structure and the amount of metrics and data displayed on my screen.

This inspired me to answer the unasked question: *“How can new Splunk users quickly learn to build meaningful dashboards?”* I believe this question was not widely addressed because the platform was still relatively new to many users, and only a small number of people had enough experience to confidently navigate it.

A signal that this resource had impact was that my presentation was included in onboarding documents for new interns. This gave individuals a reference when creating Splunk dashboards and helped them reproduce similar work more independently.
